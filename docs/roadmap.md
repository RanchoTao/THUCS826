# Preparation Roadmap

This roadmap is a practical workflow for using the repository. It does not claim to be an official Tsinghua preparation plan.

## Phase 1: Foundation rebuilding

- Goal: rebuild core CS foundations and identify missing prerequisites.
- Main tasks:
  - Review data structures and basic algorithms.
  - Revisit computer organization, operating systems, and computer networks at a survey level.
  - Audit available materials before deep study.
- Output:
  - Personal baseline checklist.
  - Initial weak-topic list.
- Success criteria:
  - Can map each available material to a subject area.
  - Can identify which materials are directly useful and which are only historical references.

## Phase 2: Core subject review

- Goal: complete a first systematic pass through the core subjects.
- Main tasks:
  - Data structures and algorithms: concepts, implementations, and problem patterns.
  - Computer organization: number representation, CPU, memory hierarchy, I/O.
  - Operating systems: processes, memory, files, synchronization, deadlocks.
  - Computer networks: layered model, IP, TCP/UDP, routing, application protocols.
- Output:
  - Subject notes.
  - Topic-by-topic problem index.
- Success criteria:
  - Each core subject has a reviewed topic map and a tracked problem set.

## Phase 3: Past-exam analysis

- Goal: understand what the available historical materials can and cannot prove.
- Main tasks:
  - Inspect 2017–2020 912 recall files.
  - Extract only verifiable metadata and accessible content.
  - Compare historical material coverage with the inferred subject map.
- Output:
  - Year-by-year exam notes.
  - Known gaps and collection targets.
- Success criteria:
  - No unsupported claims are made from filenames alone.
  - Every extracted point is traceable to a file.

## Phase 4: Mock exams and timed practice

- Goal: convert knowledge into exam performance.
- Main tasks:
  - Use past materials and selected exercises for timed practice.
  - Record mistakes by subject and topic.
  - Build answer templates where appropriate.
- Output:
  - Timed-practice logs.
  - Error notebook.
- Success criteria:
  - Can complete practice sets under time constraints.
  - Repeated errors decrease across review cycles.

## Phase 5: Final consolidation

- Goal: stabilize recall, speed, and exam strategy.
- Main tasks:
  - Review high-frequency and weak topics.
  - Revisit official admissions and exam notices.
  - Finalize material list and stop large-scale collection unless a gap is critical.
- Output:
  - Final checklist.
  - Condensed notes.
- Success criteria:
  - Key formulas, algorithms, and concepts are reviewable in a compact format.
  - Remaining uncertainties are documented rather than hidden.
